Upgrade spec (copy to Replit agent)

Add to Resource model:
	•	capacity INTEGER DEFAULT 10
	•	opening_hours JSON (e.g. {"mon":[["09:00","18:00"]], "tue":[["09:00","18:00"]], "wed":[["09:00","18:00"]], "thu":[["09:00","18:00"]], "fri":[["09:00","18:00"]], "sat":[["10:00","14:00"]], "sun":[]})

New table Pass:

id PK, email TEXT INDEX, pass_type TEXT CHECK(pass_type IN ('day','week','month')),
stripe_session_id TEXT, purchase_dt DATETIME, valid_from DATETIME, valid_to DATETIME,
status TEXT CHECK(status IN ('active','expired','refunded')) DEFAULT 'active'

Booking validation (server):
	1.	Opening hours check: chosen date/time must be within a window in opening_hours[weekday].
	2.	Capacity check: for the (resource, time window), count overlapping bookings with status in ('reserved','paid','checked_in','free'). Must be < capacity.

Pass logic:
	•	If user has an active pass (valid_from ≤ start_dt ≤ valid_to), skip Stripe; booking status='free', amount_cents=0.
	•	If no pass: proceed to Stripe like now.
	•	Pass purchase creates a Pass row; webhook sets active + valid_from/valid_to:
	•	day = midnight→23:59 same day
	•	week = 7 days from purchase (or next Monday–Sunday; pick one and document)
	•	month = 30 days from purchase

UI updates:
	•	On /book:
	•	Show capacity badge per resource: e.g., “Desks left today: 7/10” (compute for selected slot too).
	•	Radio: [ Book by the hour ] or [ Buy a pass (Day / Week / Month) ]
	•	If pass selected → Stripe Checkout for pass products; on success, show “Pass active until {date}”.
	•	When user enters email, if they have an active pass, show “Pass active (free bookings until {date})”.

Stripe products (create or let agent create):
	•	Products: EasyDesk Day Pass, EasyDesk Weekly Pass, EasyDesk Monthly Pass
	•	Prices (example): Day $10, Week $30, Month $120 (adjust later)
	•	Webhooks: handle checkout.session.completed for both booking sessions and pass sessions (differentiate via metadata.type='pass'|'booking').

Admin enhancements:
	•	/admin?key=...:
	•	Edit capacity & opening hours per resource (simple JSON textarea or small form per weekday).
	•	List active passes (email, type, valid_to) + manual expire.
	•	Toggle: “Allow pay-on-arrival” (existing) and “Require account for pass use” (optional).

Server helpers (pseudo):

def is_in_hours(resource, start_dt, end_dt): ...
def seats_left(resource_id, start_dt, end_dt):
    used = Booking.query.filter(...overlap...).count()
    cap = Resource.query.get(resource_id).capacity
    return max(0, cap - used)

def active_pass(email, at_dt):
    return Pass.query.filter_by(email=email, status='active')\
        .filter(Pass.valid_from <= at_dt, Pass.valid_to >= at_dt).first()
